# Education Register — セミナー／教育レジスター（intent gate と反転表）

`meta.intent` でデッキのレジスターを宣言する（deck-strategy Step 0）：

- `financial` | `board` — 意思決定の場。**抑制＝信頼**。失敗モードは「売り込みに見える」。
- `seminar` | `education` — 学習の場。**理解・定着・共感**。失敗モードは
  「抽象的で退屈で忘れられる／間違って覚える」。

守るべきものが逆なので、床規則のいくつかは education で**反転**する。
ただし **honesty は決して反転しない**——教育には教育固有の嘘のつき方
（無い構造を見せる／歪める圧縮／盛るインパクト）があり、財務の
honesty override を**軸別に特化して継承**する（§2）。

## 1. 反転表（intent = education 時のみ切替）

| 項目 | financial（既定） | education（反転後） |
|---|---|---|
| 図解の既定 | 迷えばテキスト | **迷えば構造を見せる**（ただし §2-1 のガード下） |
| トーン | 静か・抑制 | にぎやか可（playful-pop / メリハリ許容） |
| 1スライド1強調の軸 | 強調総量の制限 | **認知負荷の設計**（反復・コールバック・章扉の現在地表示を許容。**1スライド1強調・1デッキ1 peak は維持**） |
| 人型デバイス・吹き出し | **禁止（register-gate ERROR）** | 許可（§3 の persona 床規則下） |

## 反転しない共通の床（両レジスター常時 ON）

- **honesty override**：誘導的・誤認させる強調をしない。強調は事実に奉仕する。
- **Color Universal Design**（visual-psychology.md §2.5）：非色相チャネルの冗長化。
- **数値＋単位アトムの分割禁止**（shrink 優先、割らない）。
- **AI 画像不使用**・実在人物/IP/ロゴ不可（許諾画像枠 §3-3 を除く）。
- **AI 臭い装飾の禁止**（タイトル下線・アクセント縦帯・カード片側ボーダー等、
  house-quality-bar §2）。

## 2. 三モード（教育の内容ロジック）と軸別 honesty ガード

### 2-1 ロジカルシンキング — 構造を可視化する
**ガード：「無い構造を見せない」。** 順序が無いのに flow、相関を独立2軸に
見せる matrix、相関を因果に見せる矢印は嘘であり、**間違ったモデルを教える
ので退屈より悪い**。分類テスト：**構造を1語で言えるか**
（sequence / loop / two-axis / **positioning / system / relation /
identity（恒等式・内訳・構成） / bridge（増減） / crossing（損益分岐）**）。
言えなければ図解せずテキスト。型の根拠は notes に残し、事実・財務系は
人が承認してから確定（over-diagram lint が notes の記録を WARN で促す）。

骨格（既存 flow / cycle / matrix に追加）：
- `identity` — 積み上げ恒等式（全体＝部分。1パートは2段目に分解可＝STRAC形）。
  **和の内訳で大きさが意味を持つなら formula ではなくこちら**（額装の禁止）。
- `breakeven` — 損益分岐点図（用語のみ・データ無しは※模式図が自動）。
- `positioning` — 競争ポジション（2-3 列 + VS）。
- `system` — エコシステム／全体像（ボックス + **ラベル付き矢印**で流れ、
  戻りの流れは下段レーン）。
- `relation` — 関係マップ（左右の要素を線で対応付け）。

### 2-2 サマライジング — 高密度を消化可能に
**ガード：「歪める圧縮をしない」。** 意味を変える但し書きを落とさない
（「理想時に最大30%」を「30%速い」にしない）。要約は原典に対して擁護可能で
あること。圧縮の妥当性は人承認領域として notes に残す。

### 2-3 ビジュアライジング — 記憶に残る見せ場
**ガード：「盛らない」。** チェリーピックした巨大数字・分母を外した数字・
データ以上を匂わせる図の禁止。強調は事実に奉仕する（サンリオ p8 で
751 予測バーより 518 実績を濃くした override の一般化）。
実装既定：**写真ゼロの SVG ネイティブ**（数字リビール・大型タイポ・
コード描画・ピクトグラム）。ダーク面（サンドイッチの見せ場）可。
実写が要る場合のみ §3-3 の許諾画像枠。エンジンは画像を生成も取得もしない。

## 3. 人型デバイス（persona — asset 非依存スロット）

`content.persona = { quote, pose?, tailSide?, variant?, mark, figure? }`。
合成（吹き出し＋配置＋※例マーキング＋register gate）は**素材ソースに
依存せず不変**：

- **(a) 既定＝エンジン内 SVG**（recipes.figureSvg）：二色フラット
  （ティール線画＋ほぼ白の顔）・腰上・正面。pose を内容の感情に合わせる：
  `concern`（眉ハの字・口水平・ジェスチャー無し）→ フック・問題提起、
  `present`（片手挙げ＋笑顔）→ 解決・まとめ。髪型・上着・ネクタイで個体差。
- **(b) 差し込み＝ユーザー提供の許諾 PNG**（`figure` にパス）。エンジンは
  スクレイプも AI 生成もしない。素材の権利はユーザー責任。差し込み時も
  吹き出し・配置・※例・register gate は不変（入れ替えても崩れない）。
- 吹き出しは **seam の無い単一パス SVG**（角丸＋tail を1本のパス）を PNG 化し、
  その上に **native text**（crisp・編集可能）。tail は人物側を指す
  （`tailSide: bottom | left`）。

**persona／捏造ガード（常時 ON・床規則）**
- 吹き出しの言葉は**例示**。実在の名前ある人物に帰属させない。
- ペルソナ・説明用数値は「※学習用の架空の例」等で明示（`mark` —
  persona-mark lint が欠落を WARN）。**忠実度が上がるほど（＝許諾 PNG ほど）
  マーキングを強める**。
- `intent = financial | board` では人型・吹き出しは**常時 OFF**
  （register-gate lint が ERROR）。

## 4. イメージ喚起変換層（imaginability — education のみ ON）

レイアウトの**前段**に置く内容変換：具体化（抽象→特定インスタンス）／
類推（未知→既知）／物語化（because→then→so）／Before-After／
ワークト・エグザンプル（完成例1件）／曖昧量詞→具体数。

**ガード：** 変換で生じる具体（架空ペルソナ・説明用数値）は**例示として
マークし、事実主張と混ぜない**。例示数値は導出過程を明示
（固定費 ÷ 限界利益率 ＝ 損益分岐点）し、実在の統計に見せない。
`intent = financial` では常時 OFF。

## 5. スライドパターン（教育の棚）

| パターン | モード/要素 | 要点 |
|---|---|---|
| hook | visualizing + persona + 問い | 悩みの生の声を吹き出しで具体化＝自分事化。誇張数字は置かない |
| logical | ロジカル骨格 + 1強調 | flow/positioning/system 等。型が意味に合うか人承認（notes） |
| case | persona + ワークト + ※例 + 比較カード | 架空例に ※例 明示。1強調（注目カード） |
| big-number | visualizing（ダーク・SVGネイティブ） | 大型数字は例から導出。数字アトム非分割 |
| summary | サマライズ + モチーフ反復 | 1点1メッセージ |
| before-after | 再フレーム（専用パターン） | 誤解→正解の対比。歪めない |

各スライドは notes に設計根拠（なぜこの型か・honesty の根拠）を残す。

## 6. 正直な残差（この文書が保証しないこと）

- SVG 人物の**美観**（比率・手の自然さ）は人の目視領域。機械はレンダが
  崩れないことまで。
- 字面・游ゴシックのウェイト序列は実機 PowerPoint 依存（soffice QA の範囲外）。
- 差し込みモードの品質は許諾セットが入って初めて確定。
- 「構造を1語で言えるか」「圧縮が擁護可能か」「盛っていないか」の
  **意味判定は機械化できない**——lint は記録と印の存在まで、判定は人。


### 人物素材の出所（確定ポリシー）

- **本番の人物 = CC0部品を repo に vendor（実行時 外部非依存が大原則）。**
  - **V1（既定・最も独立）**: CC0の完成SVGを `assets/generated/figures/` にコミット。
  - **V2（オフライン量産）**: vendored npm（`@dicebear/core`+`@dicebear/collection`、
    lock固定）から `bin/graphics/gen-figures.js` が**固定シードで決定論生成**し、
    生成物をコミット（＝V1相当になる）。既定セットは `figures/openpeeps/`
    （Open Peeps, CC0 1.0 — 半身バスト。服色はテーマ accent に再着色可＝層②）。
  - **HTTP API（api.dicebear.com 等）は使わない**：ネット必須・先方障害で停止・
    出力が揺れて無回帰を壊す。
  - どの経路でも `LICENSE.md` に出所・ライセンス・帰属要否・商用可否を記録
    （実在人物の肖像・IP・ロゴ・出所不明スクショのトレースは不可）。
  エンジンは人物を AI 生成もスクレイプもしない（M-7）。
- **機内図形は非常用 fallback**（本番人物にはしない）：`style:"silhouette"`
  （ビジネスシルエット・顔なし実比率）／`style:"pictogram"`（抽象バスト）を
  明示 opt-in した場合のみ描画。未指定＋`figure` 無しは吹き出しと引用だけが
  残り、lint（PERSONA-FIGURE）がポリシーを警告する。
- **1プロジェクト1様式**：openPeeps・シルエット・供給素材を同一デッキに
  混在させない（床①の一貫性と同じ理由）。
- **persona スロットは asset 非依存・座標不変**（SVG版＝差し込み版で合成完全同一、
  機械証明済み）。素材を差し替えても吹き出し・※例・配置は崩れない。
- **シーン記号**（`persona.symbol`: worry/idea/sweat/question/up/down ＝
  悩み雲・電球・汗・？・↑↓）は人物とは別レイヤーの機内 SVG オーバーレイ。
  どの人物素材の上にも同じ規則で合成される。
