# icon-catalog — 統治されたピクトグラムの選択機関

**アイコンの定義（M-8）：役割マーカー。** ノードが「誰／何」かを名指しする時だけ
使える。数字・ムードの装飾には使えない。文字記号（✕ → ＝ 数字）で足りる情報に
絵を使わない（visual-psychology §3.5）——絵アイコンの既定は常に「使わない」で、
使う理由を注記できる時だけ使う。

## 選択規則（この順で判定）

1. **ノードはACTOR（人・組織）かOBJECT（モノ・書類・カネ）か？** どちらでも
   なければアイコン不可（概念・動作・感情は文字が運ぶ）。
2. **1スライド内で意味が排他か？** 同じアイコンが2つの違う意味を持つなら全部外す。
3. **レジストリにあるか？** 無ければ (a) 自前SVGのパス指定
   （`icon: "assets/icons/x.svg"`）、(b) レジストリへの追加提案（engine-proposal）、
   (c) 使わない、の順。**その場しのぎの絵文字・クリップアートは禁止**（lintが弾く）。
4. **色はトークン**：レジストリ品はエンジンがink色で着色。自前SVGも単色・
   テーマ整合が条件（多色クリップアートは house-quality-bar §2 違反）。

| カネ・保険料・売上 | `coin` | 保険料, 融資, キャッシュ |

## 対応パターン（v1）

- `system` のノード：`{ "label": "経営者", "icon": "person" }`（文字列ノードと混在可）。
  高さ床はアイコン分の字域シフトを同じ数式で測る（sysNodeSpec — 単一ソース）。
- `stat-grid` / `message` の既存 icon スロット：従来どおりパス指定（自前SVG）。
- 拡張（flow ノード等）は engine-proposal で。**エンジンを迂回した手貼りは禁止**（M-7）。

## 自前SVGテンプレートの持ち込み

プロジェクトの `assets/icons/*.svg` をパスで参照すれば今日から使える。恒久化
（レジストリ収載＝名前で呼べる・テーマ着色）したいものは、単色line化した上で
engine-proposalへ。多色・グラデ・写真調は収載不可。
