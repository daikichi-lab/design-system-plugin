# text-to-diagram — 原稿の文から図解への翻訳例集

図解の翻訳は「文→構造1語→骨格→content」の4段で行う。以下は判定が分かれ
やすい実例。**迷ったら記号消去テスト**（＝＋→を消しても関係が読めるか）。

| 原稿の文 | 構造1語 | 骨格 | 要点 |
|---|---|---|---|
| 「持ち物から借金を引いた残りが正味の財産」 | 恒等式 | `identity` | 面積が式。protagonist=残り。formulaにすると額装 |
| 「売上は変動費と粗利に分かれ、粗利から固定費を引くと利益」 | 恒等式×2段 | `identity`+`sub` | STRAC形。縦計算をlintが検算 |
| 「売上＝客数×客単価×店舗数」 | 因数分解 | `formula` | ×は用語関係。大きさの主張なし |
| 「固定費を粗利で回収し切った点から先が利益」 | 交差（分岐） | `breakeven` | 用語のみ。数値はデータがある時だけ |
| 「前期100が、増収+30と原価増▲12で当期118に」 | 増減（橋） | `waterfall` | レベルはtotal:true。差分は合算が合うこと |
| 「申込→審査→契約→納品の順で進む」 | 順序 | `flow` | 順序が意味を持つ時だけ。特徴の羅列はflow不可 |
| 「経営者・FP・税理士・銀行の間で情報とカネが流れる」 | 全体像 | `system` | ノードは役割。人型アイコン可（icon-catalog） |
| 「効果と工数の2軸で施策を仕分ける」 | 2軸 | `matrix` | 軸が本当に独立か検証 |
| 「保有資格者は税理士8万人、FPは173万人」 | 規模比較 | `chart` bar | 実寸比例。主役だけaccent |
| 「相談相手がいない127万社」 | 単一の数字 | `message`+stat | 図ではなく数字が主役（statTone=warn/accent） |
| 「難しそう…と感じますよね」 | 声・共感 | `dialogue` | 図解ではなく説得装置。learner系レジスター限定 |
| 「教育・住宅・老後の3大資金」 | 並列 | `stat-grid`/`card-grid` | 並列に図は不要。数えられる形で十分 |

**アンチパターン（額装）**：「A＝B＋C」を同サイズ3箱＋記号で描く。内訳の大きさが
論点なら常に `identity`。formulaの「+」を使うとdesign-lintがINFOで誘導する。
