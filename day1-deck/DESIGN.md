## 2. デザイン言語とブランド（仕様書の色表 — 一字一句コピーが原則）
```
NAVY   1B2A4A  主役。視覚重量60-70%。見出し・ダーク背景・図の主ボックス
GOLD   C9A227  道しるべ。※実装は下記 Known Gaps 参照
GOLDL  E4CE7A  ダーク面上のゴールド            → theme accentSoft / accentOnDark
VERM   C7431D  朱。痛み・代償・覚悟の1用途限定    → theme warn
VERM_D E8603A  ダーク面上の朱                  → theme warnOnDark
TINT   F3F5F9  通常カード背景                  → theme surface
TINTG  FAF5E4  「答え側」カード背景             → theme surfaceAccent
INK    2B3345 / MUTE 6E7787 / LINE_L D8DDE6
ダーク面: FFFFFF / E4E9F2 / C7D0E2 / AEB9CE / 8FA0BE
```
- 色は表から選ぶ。表に無い色が必要になったら**表の改訂案件（人間承認）**。
- ゴールドは1ページ1〜2箇所。朱の使用ページは全体の15%以下（lint WARN-TONE）。
- 本文をINKより淡くしない。
- theme.json: `day1-deck/theme.json`（13.333×7.5 座標系。仕様書の10×5.625系の
  寸法・ptは×1.3333で換算済み — タイトル27→36pt、結び帯 y4.4-4.8→5.87-6.40 等）

## 4. 常設の正直さ
- 統計値（8.2万/173万/127万社など）は原稿記載を使用し、**納品時に出典年次の確認を一言添える**
- ダミー数字も縦計算を合わせる（PL/BS/waterfall は検算済みの値のみ）
- グラフは実寸比例・単位統一・▲表記

## 5. 構成の文法（このプロジェクトの座標系）
- 情報=白 / 感情=紺。ダーク対象=表紙・章扉・締め・転換点。上限25-30%、章扉→直後1枚以外の
  連続は原則回避（どん底2連 P9-P10 は意図的例外・notes宣言）
- 章扉= `sectionStyle:"chapter"`（透かし数字なし・CHAPTER N）
- 純資産・自由なお金・答え側=常にGOLD系/右。個人=左。結び1行=全ページ同じ高さ（closing帯）
- 版分岐: 信頼シフト版固有=P10(革靴)・P21(有料開示)。本流v5はこの2ブロックを差し替え/削除

## Known Gaps（正直な記録）
1. **GOLD C9A227 は白文字とのコントラスト2.4:1で、白文字を載せる用途（表ヘッダ・
   強調ノード・数字丸）の4.5:1床を割る**。仕様書自身の規定（コントラスト調整は表の
   中から/改訂案件）に従い、ライト面の実装ゴールドは 8F7010（4.67:1）・テキスト用
   deep 7E6312 を暫定採用。ダーク面は GOLDL E4CE7A で仕様どおり。→ 表の改訂として
   人間の承認待ち。
2. 仕様書の本文行間1.2-1.3はJP本文の床（1.5-1.7）と衝突 → 床を優先（1.55）。
3. ✕マークの朱着色（カード見出し内の部分着色）は未実装 — 見出しはink色の✕。
4. 大数字76pt級（換算101pt）はstat=88ptで運用中（messageLとの序列維持のため）。
