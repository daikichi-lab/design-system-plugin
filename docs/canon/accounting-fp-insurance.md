# 正準形カタログ — 会計・FP・保険（domain canonical forms）

> **置き場所の意図：** これはドメイン知識であり、プラグイン（汎用エンジン）には
> 組み込まない。プラグインが持つのは汎用の骨格（identity / breakeven /
> waterfall / chart …）だけで、「この分野のこの概念にはこの型」という対応表は
> 利用側リポジトリの `docs/canon/` に置く（DESIGN.md と同じ思想）。
> 生成時は `deck-strategy` がこのファイルを読み、ビート→パターンの写像を
> この分野の実務の型から始める。新しい分野のカタログは
> `/pptx-creation:domain-canon` スキルで作る。

**採用ガード（全行に適用）**
1. 実務・教科書で実際に使われている型のみ（発明しない）。
2. 記号消去テストに合格する型のみ（visual-psychology §3.5 — 額装は正準形ではない）。
3. honesty列は必読 — 比例・位置・面積はデータがある時だけ意味を運ばせる。

## 会計（経営者向け・セミナー）

| 概念 | 正準形（実務の呼び名） | 骨格/パターン | honesty・容量の注意 |
|---|---|---|---|
| BS（貸借対照表の構造） | 積み上げ恒等式の箱図 | `identity` | 比例は全パート数値時のみ。負債超過は `waterfall` へ |
| STRAC／お金のブロックパズル | 2段の積み上げ恒等式 | `identity`＋`parts[].sub` | 入れ子は1パートまで。縦計算はlintが検算 |
| 損益分岐点（CVP） | 交差する2直線＋BEP | `breakeven` | 用語のみ・数値ラベル禁止。データ無しは※模式図が自動 |
| 利益ブリッジ／増減要因 | 滝グラフ | `waterfall` | ▲表記はエンジン任せ。8超は「その他」に集約 |
| 売上の分解（客数×単価） | 因数分解 | `formula` | ラベルは名詞句。文章を入れない |
| 5段階利益（売上→純利益） | 段階の滝 | `waterfall`（totals活用） | 各levelにtotal:true |
| 財務3表のつながり | 相互の流れ図 | `system` | 矢印ラベル＝何が流れるか（カネ/情報） |
| 勘定科目の対応・分類 | 対応マップ | `relation` | 分類ならzone表示（線の網にしない） |
| 月次・四半期推移 | 縦棒/折れ線 | `chart` column/line | 軸ゼロ基点・ラベルが真実（軸歪曲禁止） |
| セグメント構成比 | 100%帯 | `chart` band | 四捨五入注記を`note`に |
| 規模・件数の比較 | 実寸横棒 | `chart` bar | 実寸比例（印象操作しない） |
| 決算書のイメージ提示 | 書類風ミニチュア | **GAP（未実装）** | 現状は使わない/写真不可。要engine提案。ダミーでも縦計算を合わせる |
| 資金繰りサイクル（サイト） | 期日の帯/年表 | `timeline` | 日付は1行 |

## FP（個人・ライフプラン）

| 概念 | 正準形 | 骨格/パターン | 注意 |
|---|---|---|---|
| 額面→手取りの分解 | 積み上げ恒等式 | `identity` | 「自由なお金」をemphasisに（残りが主役） |
| 家計の収支＋自由なお金の内訳 | 2段恒等式 | `identity`＋sub | STRACの家計版 |
| ライフイベント年表 | 年表 | `timeline` | 3–7マイルストーン |
| キャッシュフロー表 | 精緻な表 | `table` | 単位・前提を`note`に。6行超は分割 |
| 資産推移シミュレーション | 折れ線 | `chart` line | 前提を明記、試算は「（試算）」打刻を著者側で |
| リスクの4象限（頻度×影響→保険/貯蓄/回避/低減） | 2×2 | `matrix` | 相関を独立2軸に見せない（軸が本当に独立か） |
| ポートフォリオ構成 | 円/帯 | `chart` pie(≤5)/band | 5切片まで |
| 3大資金（教育・住宅・老後） | 並列カード | `stat-grid`/`card-grid` | 金額は出典・前提つき |
| 公的年金の階層（1〜3階） | 積み上げ | `identity`（値なし等分可） | 階層＝構成。比例は制度額がある時のみ |

## 保険

| 概念 | 正準形 | 骨格/パターン | 注意 |
|---|---|---|---|
| 保障の全体像（公的＋企業＋私的） | 層の積み上げ | `identity` | 「私的で埋める層」をemphasisに |
| 掛金の内訳（保障＋積立） | 恒等式 | `identity` | 商品実額があれば比例 |
| 必要保障額の逓減（三角形） | 逓減ステップ／面グラフ | **GAP（未実装）** | chartにarea/step無し。暫定は`waterfall`（年代を段階に）か`timeline`＋文で。要engine提案 |
| 見直し前後 | 対比 | `comparison`/`before-after` | 有利側を右・誇張しない |
| 商品・プラン比較 | 表 | `table` | 保険料は前提（年齢・期間）必須 |
| 給付金の流れ（誰から誰へ） | 流れ図 | `system`/`flow` | 矢印ラベル＝カネの名前 |
| ライフステージ×必要保障 | 年表/2軸 | `timeline`/`matrix` | 段階なら年表、独立2軸ならmatrix |
| 加入率・世帯統計 | 実寸横棒 | `chart` bar | 出典年次を`note`に |

## GAP台帳（エンジン提案待ち）

| 型 | 用途 | 提案の方向 |
|---|---|---|
| 書類風ミニチュア | 「これ読めますか」演出（決算書PL/BS） | 新パターン `document-mock`：白地・細罫・グレー数字。数値はエンジンが縦計算を強制 |
| 逓減保障（三角形/面） | 必要保障額×収入保障のカバー | `chart` に area/step 追加 or 専用骨格 `coverage` |

GAPは手描き回避（M-7）——実装されるまで代替パターンか本文で語る。
エンジン提案は diagram-recipes.md のレシピ形式（yaml＋capacity＋floor＋look＋
Verified条件）で起票する。
